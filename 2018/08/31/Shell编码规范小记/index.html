Shell编码规范小记

Posted on 2018-08-31 | Edited on 2018-09-03 | In Bash

什么时候使用shell

  • 代码小于100行时才使用shell

头部和长度

  • 开头形如

  • 行的最大长度为80个字符。

    管道

  • 如果一行容不下整个管道操作，那么请将整个管道操作分割成每行一个管段。

  • case语句

引用

  • 变量引用。推荐用 ${var} 而不是 $var ；除非绝对必要或者为了避免深深的困惑，否则不要用大括号将单个字符的shell特殊变量或定位变量括起来。
  • 除非需要小心不带引用的扩展，否则总是引用包含变量、命令替换符、空格或shell元字符的字符串。
  • 推荐引用是单词的字符串（而不是命令选项或者路径名）。
  • 千万不要引用整数。
  • 注意 [[ 中模式匹配的引用规则。
  • 请使用 $@ 除非你有特殊原因需要使用 $* 。

    命令替换

  • 使用 $(command) 而不是反引号。

    test，[和[[

  • 推荐使用 [[ … ]] ，而不是 [ , test , 和 /usr/bin/ [ 。

    测试字符串

文件名的通配符扩展

  • 当进行文件名的通配符扩展时，请使用明确的路径

    Eval

  • 避免使用eval

    函数名

  • 使用小写字母，并用下划线分隔单词。使用双冒号 :: 分隔库。函数名之后必须有圆括号。关键词 function 是可选的，但必须在一个项目中保持一致。

    变量名

  • 同函数名

    常量和环境变量名

  • 全部大写，用下划线分隔，声明在文件的顶部。

    使用本地变量

  • 使用 local 声明特定功能的变量。声明和赋值应该在不同行。

    函数位置

  • 将文件中所有的函数一起放在常量下面。不要在函数之间隐藏可执行代码

    主函数main

  • 对于包含至少一个其他函数的足够长的脚本，需要称为 main 的函数。
  • 对于仅仅是线性流的短脚本， main 是矫枉过正，因此是不需要的

    检查返回值

  • 总是检查返回值，并给出信息返回值。对于非管道命令，使用 $? 或直接通过一个 if 语句来检查以保持其简洁。
